# 020 Memory（睡眠/做梦）设计 v0

> **状态**：已废弃，内容整合至 [025-memory-architecture-v1.md](./025-memory-architecture-v1.md)

## 设计原则

- 记忆是“网络”，不是扁平向量库
- 记忆强度可变（强化与遗忘并存）
- 睡眠期执行离线整理，清醒期专注任务执行

## 关键概念

- `Memory Node`：事件、事实、计划片段、工具结果
- `Synapse Edge`：节点间关联，带权重与方向
- `Activation`：检索时的激活强度
- `Consolidation`：短期记忆到长期记忆的巩固
- `Dreaming`：跨节点随机游走 + 约束生成的联想

## 睡眠周期（提案）

1. `Replay`：回放近期高价值轨迹
2. `Consolidate`：强化高置信/高复用连接
3. `Prune`：衰减低价值、低访问连接
4. `Abstract`：生成中层概念节点（模式/规则）
5. `Dream`：生成假设性场景，用于次日策略备选

## 梦境机制（提案）

- 输入：近期任务节点 + 历史高权重节点
- 方法：受约束随机游走（避免无意义跳转）
- 输出：`Hypothesis Nodes`（标注来源与置信度）
- 约束：梦境记忆默认不直接作为事实执行，需二次验证

## 风险与控制

- 幻觉扩散风险：梦境节点必须带上“假设”标签
- 记忆污染风险：失败轨迹与成功轨迹应区分权重
- 复杂度风险：先做轻量图结构，再逐步扩展图数据库
